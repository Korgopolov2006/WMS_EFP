В Таблице 4 представлены входные данные, которые вводит пользователь при взаимодействии с веб-приложением системы управления складом (WMS).
Таблица 4 - Входные данные
No | Имя | Тип | Ограничение | Формат ввода | Описание
Регистрация и аутентификация
1 | Username | string | a-z, A-Z, а-я, А-Я, 0-9, _; {3, 150} | Текстовое поле | Имя пользователя для входа в систему
2 | Email | string | Валидный email; {5, 254} | Текстовое поле | Email пользователя
3 | Password | string | Минимум 8 символов, включая буквы и цифры | Поле пароля | Пароль пользователя
4 | Role | string | Одно из значений ролей системы | Выпадающий список | Роль пользователя (администратор, кладовщик, аналитик и др.)
5 | BearerToken | string | 32-128 символов | Заголовок HTTP Authorization | Токен доступа для интеграционного API
Выбор рабочего контекста
1 | BranchId | integer | ID филиала; {1, 999999} | Выпадающий список | Выбор филиала для работы
2 | WarehouseId | integer | ID склада; {1, 999999} | Выпадающий список | Выбор активного склада
3 | ZoneId | integer | ID зоны; {1, 999999} | Выпадающий список | Выбор зоны хранения
Каталог - фильтрация и поиск
1 | Search | string | a-z, A-Z, а-я, А-Я, 0-9, пробел; {0, 255} | Текстовое поле | Поисковый запрос по наименованию и артикулу
2 | CategoryId | integer | ID категории; {0, 999999} | Выпадающий список | Фильтр по категории товара
3 | BrandId | integer | ID бренда; {0, 999999} | Выпадающий список | Фильтр по бренду
4 | OEMNumber | string | a-z, A-Z, 0-9, -, /; {0, 64} | Текстовое поле | Поиск по OEM-номеру
5 | AnalogNumber | string | a-z, A-Z, 0-9, -, /; {0, 64} | Текстовое поле | Поиск по аналогу
6 | InStockOnly | boolean | true/false | Чекбокс | Показывать только позиции в наличии
Приемка
1 | ReceivingNumber | string | a-z, A-Z, 0-9, -, /; {1, 64} | Текстовое поле | Номер документа приемки
2 | SupplierName | string | Любые символы; {1, 255} | Текстовое поле | Наименование поставщика
3 | ReceivingDate | date | YYYY-MM-DD | Поле даты | Дата приемки
4 | ProductId | integer | ID товара; {1, 999999} | Выбор из списка | Товар в строке приемки
5 | QtyExpected | decimal | >= 0; до 3 знаков после запятой | Числовое поле | Ожидаемое количество
6 | QtyReceived | decimal | >= 0; до 3 знаков после запятой | Числовое поле | Фактически принятое количество
7 | StorageLocationId | integer | ID ячейки; {1, 999999} | Выпадающий список | Ячейка размещения товара
Инвентаризация
1 | InventoryDate | date | YYYY-MM-DD | Поле даты | Дата проведения инвентаризации
2 | ZoneId | integer | ID зоны; {1, 999999} | Выпадающий список | Зона проверки
3 | ProductId | integer | ID товара; {1, 999999} | Выбор из списка | Инвентаризируемая позиция
4 | QtySystem | decimal | >= 0; до 3 знаков после запятой | Автозаполнение/поле | Количество по учетной системе
5 | QtyActual | decimal | >= 0; до 3 знаков после запятой | Числовое поле | Фактическое количество
6 | Comment | string | Любые символы; {0, 500} | Текстовое поле | Комментарий по расхождению
Заказы и отбор
1 | OrderNumber | string | a-z, A-Z, 0-9, -, /; {1, 64} | Текстовое поле | Номер заказа
2 | ProductId | integer | ID товара; {1, 999999} | Выбор из списка | Товар в строке заказа
3 | QtyOrdered | decimal | > 0; до 3 знаков после запятой | Числовое поле | Заказанное количество
4 | Priority | string | LOW, NORMAL, HIGH, CRITICAL | Выпадающий список | Приоритет обработки заказа
5 | PickerId | integer | ID сотрудника; {1, 999999} | Выпадающий список | Назначенный исполнитель отбора
6 | TaskStatus | string | NEW, IN_PROGRESS, DONE, CANCELED | Выпадающий список | Статус задачи отбора/отгрузки
Задачи и комментарии
1 | TaskType | string | RECEIVING, INVENTORY, PICKING, SHIPPING, OTHER | Выпадающий список | Тип задачи
2 | AssigneeId | integer | ID пользователя; {1, 999999} | Выпадающий список | Назначенный сотрудник
3 | DueDate | datetime | YYYY-MM-DD HH:MM | Поле даты и времени | Срок выполнения задачи
4 | CommentText | string | Любые символы; {1, 2000} | Текстовое поле | Комментарий к задаче
