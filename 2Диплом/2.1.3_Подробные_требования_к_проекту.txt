2.1.3. Подробные требования к проекту

Необходимо разработать веб-приложение управления складом (WMS), реализовав в составе него следующие процессы:

• Проектирование веб-приложения:
необходимо спроектировать полноценное веб-приложение на базе Django Framework с использованием архитектуры MVT (Model-View-Template), модульной структуры по предметным областям (каталог, приемка, инвентаризация, отбор/отгрузка, задачи, отчеты, API, 3D-склад, интеграции), а также интеграцию с реляционной базой данных PostgreSQL. В процессе проектирования необходимо реализовать:
- Архитектурную схему приложения (клиент-серверная архитектура);
- Структурную схему базы данных (ER-диаграмма);
- Функциональную схему ключевых складских процессов (приемка -> размещение -> резервирование -> отбор -> отгрузка -> аналитика);
- Диаграмму классов основных сущностей (models, views, services, API endpoints);
- Схему тестирования и валидации данных.

• Подключение реляционной базы данных PostgreSQL:
необходимо подключить к веб-приложению реляционную базу данных PostgreSQL, реализующую хранение данных в нормализованных таблицах, валидацию и обработку входных/выходных данных системы. База данных должна предусматривать:
- Валидацию вводимых данных на уровне Django ORM, форм и сервисного слоя;
- Хеширование паролей пользователей с использованием встроенных механизмов Django (PBKDF2);
- Связи между сущностями и каскадные правила удаления/защиты записей согласно бизнес-логике;
- Индексирование полей для ускорения выборок;
- Транзакционную целостность данных в критичных операциях (приемка, инвентаризация, отбор, отгрузка);
- Уникальные ограничения (unique constraints) для предотвращения дублирования;
- Миграционный контур и экспорт схемы БД для документирования.

• Разработка серверной части (Backend) на базе Django:
необходимо разработать backend, реализующий функции управления складскими операциями, запасами, заказами и задачами.

Для пользователей системы (авторизованные сотрудники):
• Авторизация в системе и работа в рамках назначенной роли;
• Работа с дашбордом (операционные показатели, сводка по задачам и статусам процессов);
• Просмотр номенклатуры с поиском и фильтрацией (артикулы, OEM/analog, категория, бренд и др.);
• Просмотр карточки товара (характеристики, применимость, аналоги/кросс-ссылки, доступность);
• Работа с задачами (просмотр, детализация, комментарии, изменение статусов в рамках прав);
• Использование 3D-представления склада для визуального контроля зон и объектов хранения;
• Использование EFP-проверок для поиска данных по запчастям с асинхронной обработкой запросов.

Для кладовщиков и администраторов:
• Создание и ведение приемок, добавление/сканирование позиций приемки, контроль расхождений;
• Проведение приемки с обновлением складских остатков;
• Управление остатками по зонам и ячейкам хранения;
• Запуск и проведение инвентаризаций, фиксация фактических количеств, закрытие инвентаризаций;
• Контроль исполнения связанных складских задач.

Для менеджеров по продажам и администраторов:
• Создание и сопровождение заказов;
• Подтверждение заказов и управление их статусами;
• Контроль доступности товара и резервирования;
• Запуск процесса отбора и дальнейшей отгрузки.

Для сборщиков мелких деталей и комплектовщиков/грузчиков:
• Получение назначенных задач отбора;
• Работа по строкам отбора с указанием источника (остаток/ячейка) и факта комплектации;
• Завершение этапов комплектации и отгрузки.

Для аналитиков и администраторов:
• Просмотр аналитических отчетов по складу и заказам;
• Анализ оборачиваемости, неликвидов, ошибок отбора, прогноза спроса и эффективности персонала;
• Получение агрегированных метрик для контроля SLA и качества операций.

Административные функции (Django Admin):
• Управление пользователями, ролями и доступами к складам;
• Управление справочниками (филиалы, склады, зоны, ячейки, категории, бренды, модели);
• Управление товарными позициями и связанными сущностями;
• Управление документами приемки, заказами, инвентаризациями, задачами и отчетными сущностями;
• Поддержка ручного сопровождения интеграционных и служебных данных.

• Разработка REST API:
необходимо реализовать API для интеграции внешних систем и автоматизации обмена данными. API должен предоставлять конечные точки для:
• Проверки доступности сервиса и получения данных текущего пользователя;
• Получения справочников (бренды, категории, склады, транспортные/складские структуры);
• Получения списка и карточек товаров, включая аналоги и кросс-ссылки;
• Получения и управления приемками, строками приемки и действиями по приемке;
• Получения остатков и связанных данных по складу;
• Получения и управления инвентаризациями, строками и действиями;
• Получения и управления задачами, комментариями и мониторингом задач;
• Получения и управления заказами, строками заказа и действиями по заказу;
• Получения и управления задачами отбора и строками отбора;
• Получения аналитических отчетов в JSON-формате;
• Документирования API через OpenAPI/Swagger/ReDoc;
• Аутентификации через Bearer-токены (интеграционный контур).

• Автоматизация бизнес-логики:
необходимо реализовать автоматические процессы на уровне сервисного слоя и фоновых задач:
• Автоматическое обновление остатков и резервов при операциях приемки/отбора/отгрузки;
• Автоматическое формирование и актуализация связанных задач по этапам процесса;
• Автоматическое создание backorder-позиций при дефиците;
• Контроль переходов статусов документов и задач;
• Асинхронная обработка интеграционных запросов (Celery + Redis) с контролем состояния очереди.

• Реализация системы шаблонов (Templates):
необходимо разработать пользовательский интерфейс с использованием Django Template Language, включающий:
• Ролевую навигацию по разделам системы;
• Динамическое отображение данных и статусов операций;
• Формы с серверной валидацией и корректной обработкой ошибок;
• Интерфейсы мониторинга, отчетности и визуализации склада (в том числе 3D-модуль);
• Механизм системных сообщений/уведомлений пользователю.

• Обеспечение безопасности приложения:
необходимо реализовать механизмы защиты:
• CSRF-защита для форм и действий, изменяющих данные;
• XSS-защита за счет экранирования шаблонов;
• Хеширование паролей (PBKDF2);
• Разграничение прав доступа по ролям (login_required, role_required);
• Токенная аутентификация для интеграционного API;
• Валидация входных данных на всех уровнях обработки;
• Защита от SQL-инъекций за счет использования Django ORM.

• Оптимизация производительности:
необходимо применить техники оптимизации:
• Использование select_related()/prefetch_related() для снижения количества SQL-запросов;
• Пагинация и ограничение объемных выборок;
• Индексирование часто фильтруемых полей;
• Кэширование служебных и промежуточных данных (в том числе статусов фоновых задач);
• Выделение ресурсоемких операций в асинхронные очереди.

• Управление командами (Management Commands):
необходимо реализовать/поддерживать пользовательские команды для эксплуатационных задач:
• Подготовка и заполнение демонстрационных/тестовых данных;
• Генерация и обновление схемы БД для документации;
• Выпуск интеграционных токенов API;
• Проверка очереди и фоновых интеграционных задач;
• Специализированные сценарии наполнения данных для отчетов и мониторинга.
