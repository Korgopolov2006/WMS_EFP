# Документация: 3D-визуализация склада

## Обзор системы

Система 3D-визуализации склада реализована на Django (backend) и Three.js (frontend), обеспечивает интерактивное управление складской инфраструктурой с поддержкой нескольких складов, филиалов и разграничения доступа.

**Для повышения наглядности и удобства управления складской инфраструктурой реализована интерактивная трёхмерная модель склада на основе Three.js, интегрированная с серверной частью системы и системой разграничения прав доступа.**

## Архитектура

### Backend (Django)

#### Модели (`warehouse_3d/models.py`)

1. **WarehouseLayout**
   - Хранит геометрию склада (контур пола)
   - `floor_points`: JSON массив точек [[x, z], ...]
   - `is_layout_defined`: флаг завершения разметки

2. **StorageObject**
   - 3D-объекты хранения (стеллажи, полки, ячейки, напольные зоны)
   - Позиция: `position_x`, `position_y`, `position_z`
   - Размеры: `width`, `depth`, `height`
   - Поворот: `rotation_y`
   - Связь с логическим местом: `storage_location` (ForeignKey к `catalog.StorageLocation`)
   - Методы:
     - `has_stock()`: проверка наличия товаров
     - `get_stock_count()`: количество различных товаров
     - `get_total_stock_qty()`: общее количество товаров

#### Views (`warehouse_3d/views.py`)

1. **warehouse_3d_view**
   - Отображение 3D-склада
   - Проверка доступа через `request.user.can_access_warehouse()`
   - Загрузка объектов и информации о товарах
   - Передача данных в шаблон

2. **save_layout**
   - Сохранение геометрии склада (контур)
   - Проверка прав на редактирование

3. **save_storage_object**
   - Сохранение/обновление объектов хранения
   - Проверка прав доступа

4. **delete_storage_object**
   - Удаление объектов с проверкой наличия товаров
   - Возвращает ошибку, если на объекте есть товары

#### Права доступа

- **Просмотр (VIEW)**: только просмотр 3D-сцены
- **Редактирование (EDIT)**: разметка, добавление/удаление объектов
- **Администратор (ADMIN)**: полный доступ

Проверки выполняются на стороне Django через `WarehouseAccess`.

### Frontend (Three.js + Vanilla JS)

#### Инициализация (`static/js/warehouse_3d/editor.js`)

1. **Сцена и камера**
   - Three.js Scene с освещением
   - PerspectiveCamera с изометрическим видом
   - WebGLRenderer с тенями

2. **Управление камерой**
   - Вращение: зажатая ЛКМ + движение мыши (медленное, плавное)
   - Приближение: колесико мыши (ограничено: 10-100 единиц, без "летания")
   - Предустановленные виды: сверху (1), изометрия (2), сброс (3)
   - Ограничения вращения по вертикали (0.2 - π-0.2)
   - Курсор меняется при захвате (grabbing)

3. **Разметка склада**
   - Клики по полу создают точки контура
   - Визуальная превью линиями и маркерами
   - Минимум 3 точки для завершения
   - Автоматическое построение стен после сохранения

4. **Объекты хранения**
   - Типы: RACK (стеллаж), SHELF (полка), CELL (ячейка), FLOOR (напольное)
   - Добавление: кнопки, горячие клавиши (R/S/C/F) или контекстное меню (ПКМ)
   - Перемещение: drag & drop с прилипанием к сетке
   - Валидация: объекты не могут быть вне контура склада
   - Индикаторы товаров: зеленые кубики на объектах с товарами
   - Контекстное меню: ПКМ по объекту для редактирования/удаления

5. **Отображение товаров**
   - Данные загружаются из `config.stocksByLocation`
   - На объектах с товарами отображается зеленый индикатор
   - В панели свойств показывается список товаров с количеством

6. **Обработка ошибок**
   - При удалении объекта проверяется наличие товаров
   - Показывается предупреждение с количеством товаров
   - Блокировка удаления при наличии товаров
   - Понятные сообщения об ошибках на простом языке

7. **История действий (Ctrl+Z)**
   - Сохранение действий добавления объектов
   - Отмена последнего действия
   - Ограничение истории (50 действий)

#### UI/UX

1. **Режимы доступа**
   - Бейдж в шапке: "Режим редактирования" / "Только просмотр"
   - В режиме просмотра: блокировка кнопок редактирования, иконки замков
   - Уведомление о режиме просмотра в сайдбаре
   - Горячие клавиши скрыты в режиме просмотра
   - Визуальная сетка видна только в режиме редактирования

2. **Сайдбар**
   - Инструменты (только в режиме редактирования)
   - Камера (всегда доступна)
   - Горячие клавиши (только в режиме редактирования)
   - Управление (кнопка очистки разметки, только в режиме редактирования)

3. **Контекстное меню (ПКМ)**
   - Доступно только в режиме редактирования
   - ПКМ по пустой области: меню добавления объектов
   - ПКМ по объекту: редактирование, свойства, удаление
   - Автоматическое позиционирование (не выходит за границы экрана)

4. **Очистка разметки**
   - Кнопка с предупреждением (⚠️)
   - Требует ввода точного текста подтверждения: "ОЧИСТИТЬ СКЛАД ПОЛНОСТЬЮ"
   - Кнопка активируется только при полном совпадении
   - После очистки: автоматический выход из режима редактирования

3. **Переключение складов**
   - Выпадающий список в шапке
   - Показываются только доступные склады
   - При переключении: переход на новый склад

## Визуальный стиль (Glassmorphism)

- Полупрозрачные панели с blur-эффектом
- Мягкие тени и градиенты
- Поддержка темной и светлой темы
- Плавные переходы и анимации

## Горячие клавиши

**Доступны только в режиме редактирования:**

- **R**: добавить стеллаж
- **S**: добавить полку
- **C**: добавить ячейку
- **F**: добавить напольное место
- **Del**: удалить выбранный объект
- **Ctrl+Z**: отменить последнее действие
- **Esc**: снять выделение / закрыть контекстное меню
- **ПКМ**: контекстное меню (только в режиме редактирования)
- **Space**: переключить режим (просмотр/редактирование)

**Доступны в обоих режимах:**

- **1/2/3**: виды камеры (сверху, изометрия, сброс)

## Интеграция с системой

### Связь с каталогом

- `StorageObject.storage_location` → `catalog.StorageLocation`
- Товары хранятся в `inventory.Stock` с привязкой к `StorageLocation`
- При удалении объекта проверяется наличие товаров через `Stock.objects.filter()`

### Безопасность

- Все проверки доступа на стороне Django
- Frontend только блокирует UI, но не обеспечивает безопасность
- API endpoints проверяют права перед выполнением операций

## Обработка ошибок

1. **Удаление объекта с товарами**
   - Проверка через `has_stock()`
   - Возврат ошибки с деталями (количество позиций, общее количество)
   - Блокировка удаления

2. **Валидация размещения**
   - Проверка попадания в контур склада (`isPointInsideWarehouse`)
   - Проверка пересечений (можно добавить в будущем)

3. **Ошибки загрузки**
   - Проверка наличия Three.js перед инициализацией
   - Повторные попытки загрузки с задержкой

## Расширяемость

### Добавление новых типов объектов

1. Добавить в `StorageObject.ObjectType`
2. Добавить в `OBJECT_DEFAULTS` (размеры, цвет)
3. Добавить кнопку в UI и горячую клавишу

### Интеграция с другими модулями

- Приёмка: автоматическое размещение на объектах
- Комплектация: визуализация мест подбора
- Инвентаризация: подсветка проверяемых зон

## Технические детали

- **Three.js**: r128 через CDN
- **Django**: 5.2.10
- **PostgreSQL**: хранение геометрии в JSON
- **Vanilla JS**: без фреймворков, чистый JavaScript

## Использование

1. **Создание склада**: через админ-панель или API
2. **Разметка**: открыть 3D-склад → клики по полу → завершить разметку
3. **Добавление объектов**: 
   - Кнопки в сайдбаре
   - Горячие клавиши (R/S/C/F)
   - Контекстное меню (ПКМ по пустой области)
4. **Редактирование объектов**: 
   - Клик по объекту → панель свойств
   - Контекстное меню (ПКМ по объекту)
5. **Связь с местами**: связать `StorageObject` с `StorageLocation` через админ-панель
6. **Просмотр товаров**: товары автоматически отображаются на связанных объектах
7. **Очистка разметки**: кнопка "Очистить разметку склада" → ввод подтверждения

## Ограничения и будущие улучшения

- Нет проверки пересечений объектов (можно добавить)
- Нет автоматического связывания объектов с местами (ручное через админ)
- Нет визуализации движения товаров в реальном времени
- Можно добавить поиск объектов по коду
- Можно добавить экспорт/импорт геометрии
