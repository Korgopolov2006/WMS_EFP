## 🎯 ЦЕЛЬ

Реализовать ролевую модель, где:
- Разные роли видят разные задачи и интерфейсы
- Каждая роль выполняет свой этап workflow
- Система демонстрирует полный цикл: приёмка → хранение → подбор → отгрузка

## 📋 WORKFLOW ПО РОЛЯМ

### Сценарий 1: Полный цикл

1. **SALES_MANAGER** создаёт заказ
2. **SALES_MANAGER** подтверждает заказ
   - → Автоматически создаётся задача `SHIPPING` для LOADER
   - → Автоматически создаются `PickingTask` для подбора
3. **SMALL_PARTS_PICKER** выполняет подбор мелких деталей
4. **LOADER** выполняет подбор крупногабарита
5. **LOADER** отгружает заказ
   - → Автоматически завершается задача `SHIPPING`
   - → Остатки списываются

### Сценарий 2: Приёмка

1. **STOREKEEPER** создаёт приёмку
   - → Автоматически создаётся задача `RECEIVING`
2. **STOREKEEPER** добавляет строки приёмки
3. **STOREKEEPER** завершает приёмку
   - → Создаются остатки (Stock)
   - → Выполняются Backorder (если есть)

### Сценарий 3: Инвентаризация

1. **STOREKEEPER** создаёт инвентаризацию
   - → Автоматически создаётся задача `INVENTORY`
2. **STOREKEEPER** начинает инвентаризацию
   - → Автоматически создаются строки по остаткам
3. **STOREKEEPER** заполняет фактические количества
4. **STOREKEEPER** завершает инвентаризацию
   - → Остатки обновляются
